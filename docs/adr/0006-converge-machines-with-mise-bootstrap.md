# Converge machines with mise bootstrap

`dotf run` delegates all declarative machine state to `mise bootstrap`, driven by `[bootstrap.*]` and `[dotfiles]` sections in `files/home/.config/mise/config.toml`: system packages (Homebrew formulae and APT packages), home directory files (whole-tree copy, per-platform templates, symlinks), macOS defaults, LaunchAgents, and pinned tools. `dotf` scopes package managers per platform with `MISE_SYSTEM_PACKAGES_MANAGERS`, since mise's native brew backend would otherwise also run on Linux. Casks stay with real Homebrew in a Ruby step because mise's cask backend can't recognize casks brew installed (Caskroom `.metadata` dirs read as multiple versions) and can't install tapped casks that publish no API metadata (AeroSpace). Ruby Steps otherwise remain only for imperative work mise cannot express: the login shell (`/etc/shells` plus `chsh`), `-currentHost` defaults, immutable-flag file protection, Spotlight/Time Machine/duti/AppleScript configuration, non-admin macOS formula installs via the private Homebrew prefix (mise's brew backend only targets `/opt/homebrew`), APT sources for desktop apps that can't install in containers, and update notices. `bin/bootstrap` covers the pre-mise gap (Homebrew itself, Debian prerequisites and APT sources needed by declared packages, mise itself, and seeding the global config) and hooks under `[bootstrap.hooks]` bridge ordering constraints such as lifting immutable flags before the dotfiles phase and writing `$HOME`-dependent defaults.
